# User stories
### Format
- As {user role}, I want to {perform an action} so that {I can achieve a goal}
- Acceptance criteria:
	- Given [...], when [...], then [...]

### INVEST Principles
- **Independent**
	- Self-contained (don't depend on other user stories)
	- If they cannot be rewritten to be independent, make the depende[]()ncy explicit
- **Negotiable**
	- Leaves room for design decisions
	- Don't assume technologies or aspects that the user wants
- **Valuable**
	- Delivers value by itself
- **Estimable**
	- Not too vague, developers can estimate how long it takes to implement
- **Small**
	- Cannot be split up into smaller independent valuable user stories
	- Developer can complete it in 3-4 days
- **Testable**
	- Pre-written acceptance criteria that define what fits the requirements and what doesn't
